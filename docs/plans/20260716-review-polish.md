# HomeTrap — фінальне полірування за повторним рев'ю

## Overview

Повторне візуальне рев'ю після виконання плану вирівнювання стилів (усі екрани,
light + dark, порівняння з макетом) підтвердило: базові стилі приведені до макета.
Цей цикл закриває 12 залишкових знахідок. Зміни **суто frontend**.

**Функціональні / UX (пріоритет):**
1. Тема не персиститься: вибір живе лише до перезавантаження (localStorage порожній,
   `data-theme` скидається на light після F5).
2. Заблоковані поля на виставлених/оплачених рахунках (показники, курс) виглядають
   як активні інпути — потрібен явний read-only вигляд.
3. Амбер-панель «Перевірте показники» показується навіть на оплаченому рахунку —
   доречна лише для чернетки.
4. 404 на `/invoices/:id` — глухий кут: «Не вдалося завантажити рахунок» без
   «← Рахунки» і без розрізнення «не знайдено» від мережевої помилки.

**Розбіжності з макетом:**
5. Плитки дашборда без другого рядка-примітки; «Оплачено» має зелену заливку самого
   значення замість кольорової примітки під ним (не макетний патерн).
6. Management-картка на `/apartments`: рядок-сирота «Стан … Активна» через усю
   ширину, плашка статусу останнього рахунку без підпису, немає аватара.
7. Сирі ISO-дати: «Орендар з 2025-11-13», «контракт з 2025-11-13» — поруч уже є
   локалізоване «Оплачено 16 лип. 2026 р.».
8. Графіки без проміжних горизонтальних гріделей; вісь Y із сирим максимумом
   («361») замість заокругленої шкали.

**Покращення:**
9. Пропущені місяці (жовтень 2025 без рахунку) зникають з осі графіків — сезонність
   викривлюється; потрібні порожні слоти.
10. Disabled-кнопка «Завантажити» постійно видима поруч із «Додати файли» — ховати,
    поки файли не вибрані.
11. 4-та плитка дашборда «Курс НБУ» дублює чип у шапці — замінити на «Дохід за
    12 місяців» (як у макеті).
12. Плитка «Найбільша стаття» на статистиці — зробити клікабельною (перехід на
    рахунок пікового місяця).

## Solution Overview

- **Тема**: ініціалізація `localStorage.theme` → `prefers-color-scheme` → light;
  перемикач пише в localStorage і на `<html data-theme>`. Слухати зміну системної
  теми не потрібно (YAGNI) — лише при відсутності збереженого вибору.
- **Read-only рахунок**: для issued/paid показники й курс рендеряться як текст
  (той самий грид, без інпутів) — чесніше за «сірий інпут» і збігається з макетом
  перегляду; панель попереджень — лише при `status === "draft"`.
- **404 vs мережа**: API-клієнт уже розрізняє статуси (перевірити тип помилки;
  за потреби — додати статус у клас помилки клієнта). Екран помилки рахунку:
  «Рахунок не знайдено» (404) / «Не вдалося завантажити рахунок» (інше) + постійне
  посилання «← Рахунки».
- **Nice ticks**: утиліта `niceScale(maxValue)` (крок 1/2/5×10ⁿ, 3–4 поділки) для
  обох графіків; гріделі — hairline `--color-border`, підписи `--color-text-muted`.
- **Порожні місяці**: генерувати повний ряд місяців періоду (from→to), місяць без
  даних: у барах — порожній слот із міткою місяця, у лініях — розрив лінії (сегменти),
  точки лише на наявних даних.
- **«Найбільша стаття» клік**: по кліку — знайти в наявному списку рахунків
  (`getInvoices(apartment_id)`) рахунок за `peak_period` і перейти на нього;
  якщо не знайдено — плитка не клікабельна (без падіння).

## Technical Details

- Формат повної дати: `formatDate("2025-11-13")` → «13 лист. 2025 р.»
  (`Intl.DateTimeFormat("uk-UA", { day: "numeric", month: "short", year: "numeric" })` —
  той самий вигляд, що вже має «Оплачено 16 лип. 2026 р.»).
- Плитки дашборда: значення завжди нейтральним `--color-text`; примітка — окремий
  рядок 12.5px під значенням; кольорові класи `note-pos`/`note-neg` лише для
  приміток. Примітки з наявного API: «Заборгованість» — кількість прострочених
  із attention-списку (neg), «Нараховано» — кількість чернеток (з attention),
  «Оплачено» — без примітки, якщо даних немає; «Дохід за 12 міс» — «оренда +
  комунальні». Плитка без даних — без примітки (без вигаданих текстів).
- Management-картка `/apartments`: структура рядка як на дашборді (аватар, назва,
  суб-рядок орендаря), статус квартири — бейдж «Активна/Архівна» біля назви,
  плашка останнього рахунку з підписом «останній рахунок»; дії «Редагувати /
  Архівувати» — праворуч унизу; рядок-сирота «Стан» прибрати.
- Read-only значення показників у issued/paid: `formatReading`; курс — `formatRate`.
