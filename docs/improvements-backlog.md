# Беклог покращень і нових можливостей HomeTrap

Живий документ із ідеями для підвищення юзабіліті порталу. Джерело — спільний
брейнсторм на основі аналізу коду HomeTrap та огляду ринку property-management
порталів (TenantCloud, DoorLoop, Buildium, RentRedi, Baselane).

## Як користуємось

1. **Уточнення.** Проходимо ідеї одна за одною (за пріоритетом #1→#20), уточнюємо
   обсяг, заповнюємо блоки «Уточнення» та «Критерії готовності», ставимо статус
   `✅ уточнено`.
2. **Оновлення.** Після уточнення список може мінятись: пункти зливаються,
   діляться, переставляються за пріоритетом.
3. **Реалізація.** Беремо **по одній ідеї за цикл** (щоб уникати плутанини):
   уточнена ідея → окремий план (`docs/plans/`) → імплементація → перевірка →
   коміт. Наступну ідею починаємо лише після завершення попередньої.

## Легенда

- **Тип:** 🔧 покращення наявного · ✨ нове
- **Вплив:** 🔴 високий · 🟡 середній
- **Обсяг:** S (кілька днів) · M (1–2 тижні) · L (3+ тижнів) — орієнтовно
- **Статус:** 📥 беклог · 🔧 уточнюється · ✅ уточнено · 🏗️ реалізується · ✔️ готово

## Що вже реалізовано (не дублюємо)

Автозаповнення попередніх показань лічильників; облік боргів (`outstanding`) і
нагадування власнику про прострочені рахунки (Telegram/email); дашборд
`charged/paid/outstanding` + `needs_attention`; базовий графік споживання;
адаптивні media-queries; вкладення договорів; статистика простою; фільтр
статистики за орендарем; імпорт історії з XLSX; курси НБУ USD→UAH.
